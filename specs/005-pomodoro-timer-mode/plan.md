# Implementation Plan: Integrated Pomodoro Timer Mode

**Branch**: `005-pomodoro-timer-mode` | **Date**: 2026-04-06 | **Spec**: [spec.md](spec.md)
**Input**: Feature specification from `/specs/005-pomodoro-timer-mode/spec.md`

## Summary

Adds an integrated Pomodoro Timer mode to the Focus CLI. Users can run `focus start --pomodoro "task"` for a CLI-based timer loop, or use the TUI's new Pomodoro view for a full interactive experience with pause/resume/skip. Completed work phases are persisted as session records (with `mode='pomodoro'`) in the existing sessions table. A new `pomodoro_stats` table tracks daily aggregates. A new `focus pomo-stats` command surfaces per-day and per-week statistics.

Technical approach: add `src/pomodoro/` module for timer state machine + config loading; migrate DB schema additively (add `mode` column + `pomodoro_stats` table); integrate Pomodoro tab into TUI; add `pomo-stats` CLI subcommand. New dependency: `toml = "0.8"` for config file parsing.

## Technical Context

**Language/Version**: Rust stable 1.77+  
**Primary Dependencies**: clap 4, rusqlite 0.31 (bundled), ratatui 0.29, crossterm 0.28, chrono 0.4, colored 2, anyhow 1, thiserror 1, dirs 5, ctrlc 3. New: `toml = "0.8"`.  
**Storage**: SQLite WAL mode via existing `db::open_db()`. Two additive changes: `ALTER TABLE sessions ADD COLUMN mode TEXT NOT NULL DEFAULT 'freeform'`; `CREATE TABLE IF NOT EXISTS pomodoro_stats (...)`.  
**Testing**: `cargo test` — unit tests inline (`#[cfg(test)]`) and integration tests in `tests/integration/`.  
**Target Platform**: macOS, Linux (x86_64, aarch64); single static binary.  
**Project Type**: CLI + TUI tool.  
**Performance Goals**: Timer accuracy ≤ ±2s over 2-hour cycle; TUI updates every 100ms (existing poll interval).  
**Constraints**: Single binary (no daemon, no network); no tokio; offline-capable.  
**Scale/Scope**: Single user; local SQLite; ~10 new source files; ~4 modified.
